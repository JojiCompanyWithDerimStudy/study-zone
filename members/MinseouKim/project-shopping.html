공통 상단 네비게이션 바

요구사항

  • 모든 페이지의 상단에 네비게이션바가 있어야한다. 네이게이션바에는 아래와 같은 링크가 존재한다.
    • 홈 페이지 링크 (사이트 로고)
    • 로그인 페이지 링크 (로그인중일때는 로그아웃 버튼)
    • 장바구니 페이지 링크 (장바구니 갯수가 표시된다.)
    • 신상품 페이지 링크

메인 페이지

요구사항

  • 계절에 맞는 옷 정보를 확인 할 수 있다.

회원가입 페이지

요구사항

  • 사용자의 필요한 정보를 기입하여, 회원가입을 진행할 수 있다. 이때, 필요한 정보는 아래와 같다.
    • 아이디
    • 비밀번호
    • 이름
    • 나이
    • 성별
    • 주소
    • 전화번호
  • 회원가입에 성공하면, 회원가입에 성공했음을 알리고, 로그인 페이지로 이동한다.
  • 회원가입에 실패하면, 회원가입에 실패했음을 알리고, 실패 사유를 알려준다.

장바구니 페이지

요구사항

  • 장바구니에 담긴 상품들을 표시한다.
  • 장바구니에 담긴 상품이 없을경우, "상품이 없습니다" 라고 표시한다.
  • 장바구니에 담긴 상품들 중, 원하는 상품들을 선택하여 주문 할 수 있다.
  • 장바구니에 담긴 상품을 장바구니에서 제거할 수 있다.

로그인 페이지

요구사항

  • 회원가입을 통해 아이디와 비밀번호를 입력받아 로그인을 할 수 있다.
    • 로그인에 성공하면 홈 화면으로 이동한다.
    • 로그인에 실패하면 로그인에 실패했음을 알린다.

신상품 페이지

요구사항

  • 현재 새로나온 상품들만을 표시한다.
  • 카테고리 별로 상품 목록을 조회할 수 있다. 이때, 카테고리 종류는 아래와 같다.
    • 전체보기
    • 아우터
    • 상의
    • 하의
    • 악세서리
  • 상품을 선택하면, 해당 상품의 상세 페이지로 이동한다.

상품 상세 페이지

요구사항

  • 상품에 대한 정보들을 볼 수 있다. 상품 정보는 아래와 같다.
    • 제품명
    • 제품 품번
    • 제품 사진
    • 제품 설명
    • 제품 가격
    • 제품 사이즈
    • 제품 사이즈별 재고 유무
    • 제품 후기
  • 배송 / 교환 / 반품 정보를 확인 할 수 있다.
  • 상품의 사이즈를 선택할 수 있어야 한다.
  • 상품을 장바구니에 추가할 수 있다.
  • 상품 바로 구매 요청시, 결제 페이지로 이동한다.

검색 페이지

요구사항

  • 사용자로부터 검색어를 입력받아 상품을 검색할 수 있다.
  • 이때 검색은 아래와 같은 정보로 검색이 가능해야한다.
    • 제품명
    • 품번
    • 의류 종류
  • 검색 요청시, 해당 검색어에 대한 검색 결과 페이지로 이동한다.
  • 검색했던 정보를 최신순으로 최대 5개까지 알려준다.

검색 결과 페이지

요구사항

  • 검색 결과로 나타난 상품 목록을 보여준다.
  • 카테고리 별로 상품 목록을 조회할 수 있다. 이때, 카테고리 종류는 아래와 같다.
    • 전체보기
    • 아우터
    • 상의
    • 하의
    • 악세서리
  • 상품을 선택하면, 해당 상품의 상세 페이지로 이동한다.

결제 페이지

요구사항

  • 사용자로부터 주문 정보를 입력받을 수 있다. 이때 주문 정보는 아래와 같다.
    • 이름
    • 휴대전화
    • 이메일
    • 배송지 정보
  • 이미 사용자 정보로부터 알고 있는 정보가 있다면, 미리 기입해둔다.
  • 총 결제 금액을 확인 할 수 있다.
  • 결제 버튼을 눌러, 결제를 진행할 수 있다.
  • 결제에 성공하면, 결제 성공 페이지로 이동한다.
  • 결제에 실패하면, 결제 실패 사유를 알려준다.

결제 성공 페이지

요구사항

  • 결제가 완료되었음을 표시합니다.
  • 배송 관련 정보를 표시합니다.
    • 주문번호
    • 주문자명
    • 결제된 금액
    • 결제 일시
    • 카드명
    • 카드번호
